# FAIR Principles Alignment

## Findable

The manifest records file paths, checksums, source systems, dataset ids, row/column counts, and processing stages. Mapping catalogs expose source fields and target CDM fields.

## Accessible

All configured inputs come from public repositories. Download failures create explicit manual instructions instead of silently substituting fake data.

## Interoperable

The CDM uses stable table definitions, controlled vocabulary examples, source-to-target rules, and DuckDB-compatible SQL. BIDS metadata, GEO matrices, and Cell Painting profile fields are normalized into shared lineage patterns.

## Reusable

Documentation covers data inventory, mapping rules, QA acceptance criteria, contributor onboarding, processing design, and role-aligned evidence. Checksums and runtime configs support reproducible reruns.
